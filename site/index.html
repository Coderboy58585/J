Julia Hub Operator Notes

Support, Keys, And Device Locks

This page is for support information and safe license administration. Keep private keys, plaintext exports, and admin credentials off public pages.

User Commands

;hwid shows the user's HWID hash for support.

;deviceinfo shows platform, locale region, resolution, and HWID source.

;reloadkeys refreshes the public key manifest.

;reloadhwid refreshes the HWID lock manifest.

Admin HWID Lock

Ask the user for their ;hwid hash, then run:

powershell -ExecutionPolicy Bypass -File .\julia_hwid_admin.ps1 -Action Lock -Key "KEY-HERE" -HwidHash "HWID-HASH" -CommitPush

Reset a lock with:

powershell -ExecutionPolicy Bypass -File .\julia_hwid_admin.ps1 -Action Reset -Key "KEY-HERE" -CommitPush

Security Posture

The public loader is encrypted to reduce casual copying, but client-delivered code can still be inspected by determined users.

The strongest setup is a private backend that verifies key hash, HWID hash, Roblox user ID, and revocation state before returning access.

Privacy Note

The in-script device card shows platform and locale-derived region only. It does not perform IP geolocation.

Country-level checks should stay server-side in experiences you own and should be disclosed to users.

Keep this portal behind host-level access for real privacy.
